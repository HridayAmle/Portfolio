WEB DESIGN AND DEVELOPMENT

  • by:Hriday Amle
  • date:April 06, 2024
DAY 1: INTRODUCTION TO WEB DEVELOPMENT

Web development refers to the creating, building, and maintaining of websites. It includes aspects such as web design, web publishing, web programming, and database management. It is the creation of an application that works over the internet i.e. websites.

Text editors are essential tools for web development because they provide a streamlined environment for writing and editing code with features like syntax highlighting, code completion and plugin support. Popular text editors include Sublime Text, Visual Studio Code, and Atom.

Why I chose VS Code?

Visual Studio Code (VS Code) is favoured for web development due to its robust features, fast performance, cross-platform support, extensive customization options, and vibrant ecosystem, making it a top choice among developers.

The session began with an overview of essential web technologies, including HTML, CSS, and JavaScript, setting the stage for deeper exploration. we gained insights into the structure of web pages, learning about HTML tags, elements, and attributes. Then we were guided through the process of installing Visual Studio Code (VS Code) on their computers. This step ensured that everyone had the necessary tools to follow along with the coding exercises.

As an assignment, we were tasked with creating our college time table using HTML. This assignment serves as a practical application of the HTML knowledge gained during the session, allowing us to reinforce their understanding of HTML elements, attributes, and tags. We were encouraged to approach it with creativity and attention to detail.

  
DAY 2: HTML Deep Dive & CSS Kickstart

The agenda for Day 2 was focused on laying the groundwork for web development, covering essential concepts in HTML and CSS. The day's session was designed to gain more insights on document structure. The session commenced with an overview of HTML semantic markup and its importance in structuring webpages. Semantic HTML tags, play a crucial role in structuring webpage content for enhanced accessibility and search engine optimization (SEO), ensuring better comprehension by both users and search engines alike.

Following the overview, we were introduced to basic styling in CSS. We learned about CSS syntax, selectors, properties and values laying groundwork for hands-on exercises. We were encouraged to unleash their creativity and explore a variety of styling methods to elevate the visual appeal and usability. For Today’s Assignment, we were tasked with creating a HTML form containing all form attributes and styling it using CSS. Today we were given an additional assignment of designing and styling a set of buttons. This made me explore various button styles and hover effects and it was very insightful.

  
DAY 3: CSS MASTERY: ADVANCED WEB STYLING

This session began by revisiting CSS fundamentals, which helped refresh my memory and set the stage for diving into more advanced topics. Learning about CSS Flexbox and Grid layouts was particularly exciting; it gave me a whole new perspective on how to structure and organize my web pages for better responsiveness and layout control.

Next, we explored CSS Animations and Transitions. Being able to add subtle animations to elements on my website really brought them to life and made the user experience more engaging. Plus, learning about Responsive Design Strategies was invaluable. As our assignment, we are tasked with creating a resume using HTML and CSS. This presents an exciting opportunity to apply the advanced CSS techniques we've learned and showcase our design skills in a practical context.

DAY 4: JAVASCRIPT ESSENTIALS

JavaScript is a dynamic programming language that's used for web development, in web applications, for game development, and lots more. It allows you to implement dynamic features on web pages that cannot be done with only HTML and CSS.

The session began with basic introduction and syntax format of JavaScript Programming Language. Further, we explored its core concepts, data types and variables. As the workshop unfolded, I delved deeper into more advanced topics like functions, conditionals, and loops. Discovering how to create functions to streamline code and navigate conditional statements was particularly enlightening, offering me new avenues for optimizing my JavaScript projects.

For the assignment, participants are tasked with creating a World Clock using HTML, CSS, and JavaScript. The objective is to design and implement a web application that displays the current time for different cities around the world.

  
DAY 5: GIT & GITHUB FUNDAMENTALS

In this session, we delved into the fundamentals of Git, including repositories, commits, branches, and merges. Through hands-on exercises, we gained practical experience in initializing Git repositories, making changes to files, and tracking changes over time. Following are the steps to deploy a project to a repository hosted on GitHub.

  1. Initialize GitHub: If your project is not already using Git for version control, initialize a new Git repository in your project directory:
    2. git init
  2. Add Files: Add the files you want to include in the repository to the staging area:
    3. git add .
  3. Commit Changes: Commit the staged files to the local repository with a descriptive message:
    4. git commit -m "Initial commit"
  4. Create a GitHub Repository: Go to GitHub and create a new repository. Choose a name for your repository, optionally add a description, and configure other settings as needed.
  5. Add GitHub Repository as Remote: Associate your local repository with the GitHub repository by adding the GitHub repository URL as a remote:
    git remote add origin [repository_URL]
    Replace [repository_URL] with the URL of your GitHub repository.
  6. Push to GitHub: Push your committed changes from the local repository to the GitHub repository:
    git push -u origin master
    This command pushes the changes to the master branch of the remote repository named origin. If you're working on a different branch, replace master with the name of your branch.
  7. Verify Deployment: Once the push is complete, visit your GitHub repository in a web browser to verify that your project files have been successfully deployed.
